# 3D Object Creation Guide

## Procedural Mesh Generation

The Rustix engine provides procedural mesh generation for common 3D shapes through the `rustix_render::mesh::procedural` module.

## Available Shapes

### Cube

Creates a cube centered at origin with the given size.

```rust
use rustix_render::mesh::procedural;

let (vertices, indices) = procedural::cube(size: f32);
// Example: 2x2x2 cube
let (verts, indices) = procedural::cube(2.0);
```

### Quad (Grid)

Creates a subdivided quad (grid) for terrain/water.

```rust
let (vertices, indices) = procedural::quad(size: f32, subdivisions: u32);
// Example: 10x10 subdivided quad
let (verts, indices) = procedural::quad(10.0, 10);
```

### Torus

Creates a torus (donut shape).

```rust
let (vertices, indices) = procedural::torus(major_radius: f32, minor_radius: f32, major_segments: u32, minor_segments: u32);
// Example: ring with 2 unit tube, 16 major and 8 minor segments
let (verts, indices) = procedural::torus(2.0, 0.5, 16, 8);
```

### UV Sphere

Creates a UV-mapped sphere.

```rust
let (vertices, indices) = procedural::uv_sphere(radius: f32, rings: u32, sectors: u32);
// Example: sphere with 8 latitude and 16 longitude divisions
let (verts, indices) = procedural::uv_sphere(1.0, 8, 16);
```

### Icosphere

Creates a subdivided icosahedron (spherical with uniform distribution).

```rust
let (vertices, indices) = procedural::icosphere(radius: f32, subdivisions: u32);
// Example: sphere with 2 subdivisions
let (verts, indices) = procedural::icosphere(1.0, 2);
```

## Creating and Rendering Meshes

### Basic Mesh Creation

```rust
use rustix_render::{Renderer, mesh::Mesh};

// Create vertices and indices for a cube
let (verts, indices) = procedural::cube(1.0);

// Convert to bytes for GPU upload
let vertex_bytes: Vec<u8> = bytemuck::cast_slice(&verts).to_vec();
let index_bytes: Vec<u8> = bytemuck::cast_slice(&indices).to_vec();

// Create the mesh in GPU memory
let mesh = Mesh::new(
    &renderer,
    "cube",
    &vertex_bytes,
    verts.len() as u32,
    Some((&index_bytes, indices.len() as u32))
)?;
```

### Using with ECS

```rust
use rustix_render::{MeshRenderer, Transform};
use rustix_core::ecs::World;

// Spawn entity with mesh renderer
let cube_mesh = /* created mesh index */;
let transform = Transform::from_translation(Vec3::new(0.0, 0.0, 0.0));

world.spawn((
    MeshRenderer { mesh_idx: cube_mesh },
    transform,
));
```

## Vertex Format

### Standard Vertex

```rust
#[repr(C)]
pub struct Vertex {
    pub position: [f32; 3],  // World position
    pub normal: [f32; 3],    // Surface normal for lighting
}
```

- 24 bytes per vertex
- Automatically generated by procedural functions

## Custom Mesh Creation

### From Triangle List

```rust
// Create a simple triangle
let vertices = vec![
    Vertex { position: [-0.5, -0.5, 0.0], normal: [0.0, 0.0, 1.0] },
    Vertex { position: [ 0.5, -0.5, 0.0], normal: [0.0, 0.0, 1.0] },
    Vertex { position: [ 0.0,  0.5, 0.0], normal: [0.0, 0.0, 1.0] },
];
let indices = vec![0u16, 1, 2];

let mesh = Mesh::new(&renderer, "triangle", 
    bytemuck::cast_slice(&vertices).as_bytes(), 
    vertices.len() as u32,
    Some((bytemuck::cast_slice(&indices).as_bytes(), indices.len() as u32))
)?;
```

## Primitive Types Reference

| Function | Vertices | Indices | Notes |
|----------|----------|---------|-------|
| `cube` | 24 | 36 | 6 faces, 4 verts each |
| `quad` | (sub+1)² | 6×sub² | Grid topology |
| `torus` | (maj+1)(min+1) | 6×maj×min | Ring shape |
| `uv_sphere` | (rings+1)(sectors+1) | 6×rings×sectors | Latitude/longitude |
| `icosphere` | Variable | Variable | Uniform distribution |

## Performance Tips

1. **Use indices** - Reduces vertex duplication
2. **Subdivision balance** - More subdivisions = smoother but more triangles
3. **Frustum culling** - Large meshes should be split for culling
4. **LOD meshes** - Provide multiple detail levels for distance rendering
