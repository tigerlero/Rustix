Engine Architecture

A layered, modular monolith built in Rust. From ECS core to Vulkan renderer, every layer has a single responsibility and a clear dependency direction.

Layer Stack

Rustix is organized into six layers. Each layer depends only on layers beneath it, eliminating circular dependencies and enabling isolated testing.

ECS Architecture

We use hecs — a lightweight archetypal ECS (~1/10th the deps of bevy_ecs). Entities with the same component types are stored in contiguous arrays for cache-friendly iteration. System scheduling is our own code, giving full control over parallelism.

use hecs::{Entity, World as EcsWorld, Query, EntityBuilder}; // Spawn an entity with multiple components let e = world.spawn(( Transform { position: Vec3::ZERO, .. }, Name("Player".into()), Health::new(100.0), CombatStats { attack_damage: 15.0, .. }, )); // Query all living enemies for (e, transform, health) in world.query::<(Entity, &Transform, &Health)>().iter() { if health.is_alive() { /* ... */ } }

Combat System

Implemented in the runtime app as an ECS module. Supports both 2D and 3D projects with event-based damage resolution.

Combat Components

Game Loop

Variable render + physics step. Player and enemy updates run every frame. Physics applies velocity deltas and writes back transforms.

// 1. Input poll input.poll(); // 2. Player & enemy AI player::update_players(world, manager, cam, input, dt, &mut events); enemy::update_enemies(world, dt, &mut events); // 3. Combat resolution combat::tick_cooldowns(world, dt); combat::resolve_damage(world, &events); let dead = combat::cleanup_dead(world); // 4. Physics integration let results = step_physics(&mut bodies, &physics_world, dt); for (entity, pos_delta, rot_delta) in results { transform.position += pos_delta; transform.rotation += rot_delta; } // 5. Camera & animation camera.update(input, dt); update_animators(&mut animators, &clips, dt); // 6. Render submission frame_graph.compile().submit();

Editor Integration

The editor UI is built with egui inside the runtime. Combat entities can be spawned directly from the Hierarchy panel, and the inspector exposes all component fields for live editing.

✓ Create Player

Spawns a player entity (3D capsule or 2D quad) and registers it in the PlayerManager.

✓ Create Enemy

Spawns an enemy with default AI stats. Adapts to Dim2/Dim3 automatically.

✓ Undo / Redo

Every spawn is recorded as an AddEntity action in the undo history.
